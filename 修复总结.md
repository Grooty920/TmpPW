# 小杨王子博客 - 问题修复总结

## 一、概述

本次修复针对博客项目 https://lovealwayshere.cc.cd 在 Cloudflare Pages 部署后出现的多项问题，包括子页面无法访问、Service Worker 缓存异常、HTML 结构错误等。

## 二、核心问题：子页面 ERR_FAILED

### 症状

- 首页 / 正常加载，但 /blog、/tools、/about、/search 等子页面返回 ERR_FAILED
- 构建成功，dist/ 目录中所有页面文件正确存在
- 部分情况下页面 瞬间可访问，然后断掉

### 根因分析

Service Worker 预缓存机制导致。旧版 SW（v4）在安装时执行 cache.addAll() 预缓存所有页面。Cloudflare Pages 对这些 URL 的请求可能返回非 200 响应，导致 cache.addAll() 整体失败。但 SW 又调用了 skipWaiting() 立即激活，结果：

1. 第一瞬间：页面从网络正常加载，SW 同时安装
2. SW 激活后：接管页面请求，但缓存是空的（预缓存失败）
3. 再次请求：SW 从空缓存读取 → 页面中断

### 修复方案

SW v5 网络优先策略：不预缓存任何页面，每次请求先走网络，成功后缓存副本，网络失败时回退到缓存。

## 三、其他修复问题

- Layout.astro：移动端导航按钮重复ID改为 theme-toggle-mobile
- index.astro：Hero 区域移除 2 个多余闭合 div 标签
- about.astro：改用 profile.json 动态数据
- 删除 twikoo-vercel/ 和 cf-comment-worker/ 干扰目录
- admin/config.yml：分支名 master 改为 main
- package.json：移除未使用的 shuimo-ui 依赖
- 添加 public/_headers Cloudflare Pages 缓存策略配置

## 四、修改文件清单

- public/sw.js - SW v5 网络优先策略
- public/_headers - 新增缓存策略配置
- src/layouts/Layout.astro - 修复重复ID
- src/pages/index.astro - 修复多余div
- src/pages/about.astro - 改用profile.json
- public/admin/config.yml - 分支名修复
- package.json - 移除未使用依赖
- twikoo-vercel/ - 删除干扰目录
- cf-comment-worker/ - 删除干扰目录

## 五、关键教训

1. SW 的 cache.addAll() 预缓存在 Cloudflare Pages 环境下不可靠
2. 网络优先策略比缓存优先策略更稳健
3. 项目根目录不应包含无关配置文件
4. HTML 结构错误会导致浏览器解析异常
